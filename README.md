# 枪豆人（QDR）

一个基于 `Canvas + Node.js + Socket.IO` 的俯视角多人合作射击小游戏。  
核心玩法是：**通过开枪后坐力推动角色/船体移动**，在水面环境中完成波次战斗与关卡推进。

---

## 项目特性

- 单机闯关 + 联机合作双模式
- 后坐力移动（非传统 WASD 移动）
- 波次推进与多关卡主题
- 敌人类型扩展（assault / turret / charger / jammer / boss）
- Boss 三阶段机制
- 掉落与团队 Buff（散弹 / 护盾 / 快射）
- 单机波次三选一强化系统
- 受击方向提示（Hit Indicator）
- 局后战报与失败建议
- 菜单进度、成就摘要、每日挑战提示
- 联机同步优化（`stateFast` / `stateMeta` 分层 + 插值渲染）
- 低帧率视觉降级策略（不影响判定）

## 基本玩法

### 操作

- 鼠标移动：瞄准
- 鼠标左键：开火
- 角色/船体移动来自开枪后坐力（反方向推进）

### 关键机制

- 靠墙开火可获得更强反冲
- 单机每波结束可进行一次三选一强化
- 联机队伍共享 Buff（拾取后对全队生效）
- 船体颠簸过高会触发翻船惩罚（联机）

---

## 联机说明

1. 启动服务端：`npm start`
2. 打开浏览器访问 `http://localhost:3333`
3. 在主菜单输入相同房间名并进入
4. 全员点击“准备”后开局

> 如果你从本地文件直接打开页面（`file://`），客户端会尝试连接到 `http://localhost:3333`。

## 开发建议

- 调平优先看 `server.js` 中波次、敌人与 buff 时长参数
- 视觉与手感优先看 `game.js` 中粒子、镜头抖动与 HUD 逻辑
- 修改联机协议字段时，尽量保持 `state` 与 `stateFast/stateMeta` 兼容

## 常见问题

### 1) 联机无法连接

- 确认已执行 `npm start`
- 确认端口 `3333` 未被占用
- 确认浏览器可访问 `http://localhost:3333`

### 2) 画面卡顿

- 项目已内置低帧率视觉降级策略
- 可关闭浏览器其它高占用页面后再测试

### 3) 房间进不去或状态不同步

- 尝试刷新页面并重新进房
- 确认双方使用同一服务端地址与同一版本代码
